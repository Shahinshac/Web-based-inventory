# Net Profit Calculation & Expense Separation Fix

## Problem Statement

**Issue:** Net Profit was showing negative (₹-4126) even though Gross Profit was correct (₹2174).

**Root Cause:** Inventory restock costs (₹6300) were being double-counted:
1. First as Cost of Goods Sold (COGS) in profit calculation
2. Second as an "Expense" causing negative net profit

**Formula Error:**
```
Net Profit = Gross Profit - Total Cost (WRONG - double counts)
Net Profit = ₹2174 - ₹6300 = ₹-4126 ❌
```

---

## Correct Business Logic

### Cost Accounting

**Inventory Purchase (Restock):**
- ₹6300 spent on product restock
- This is NOT a business expense
- Treated as stock/asset
- Already included in product's `costPrice`

**Business Expenses (Operational):**
- Salary: ₹5000
- Rent: ₹2000
- Utilities: ₹1000
- These ARE business expenses that reduce profit

### Profit Calculation Formula

```
Step 1: Calculate per-bill profit
    base_price = selling_price / (1 + gst_rate/100)
    profit_per_item = base_price - cost_price

Step 2: Sum all bill profits (Cost already subtracted)
    Gross Profit = Total Profit from all bills
                 = ∑(base_price - cost_price)

Step 3: Subtract ONLY operational expenses
    Net Profit = Gross Profit - Operating Expenses
               = Gross Profit - (Salary + Rent + Utilities + ...)
```

### Correct Example

```
Revenue (Sales) = ₹10,000
Cost of Stock = ₹6,300
Gross Profit = ₹3,700

Operating Expenses:
  - Salary:    ₹2,000
  - Rent:      ₹1,000
  - Utilities: ₹700
  Total:       ₹3,700

NET PROFIT = ₹3,700 - ₹3,700 = ₹0 ✅
```

#### 2. **Prevent Manual Inventory Expenses**

```python
# In POST endpoint, reject attempts to create inventory-type expenses
if expense_type == 'inventory':
    return jsonify({"error": "Inventory expenses must be auto-generated"}), 400

# Always set manual expenses as 'operational'
expense = {
    "expenseType": "operational",  # Hardcoded for safety
    "autoGenerated": False,         # Mark as manual
    ...
}
```

#### 3. **Inventory vs Operational Distinction**

```
AUTO-GENERATED EXPENSES (autoGenerated = True):
  - System-created when inventory is purchased
  - Represents product restock costs
  - EXCLUDED from net profit calculations
  - Example: "Product purchase - Laptop"

MANUAL OPERATIONAL EXPENSES (autoGenerated = False):
  - User-created business expenses
  - Salary, rent, utilities, maintenance, etc.
  - INCLUDED in net profit calculations
  - Example: "Monthly staff salary"
```

### Changes to `routes/analytics.py`

#### 1. **Enhanced Operating Expense Filter**

**In GET `/api/stats`:**
```python
expenses_cursor = db.expenses.aggregate([
    {"$match": {
        "$and": [
            {"autoGenerated": {"$ne": True}},     # Exclude auto-generated
            {"expenseType": {"$ne": "inventory"}} # Exclude inventory type
        ]
    }},
    {"$group": {"_id": None, "total": {"$sum": "$amount"}}}
])
total_expenses = exp_list[0]['total'] if exp_list else 0
```

#### 2. **Correct Net Profit Calculation**

**In GET `/api/revenue-profit`:**
```python
# NET PROFIT = Gross Profit - Operating Expenses
# Cost is ONLY subtracted once (in bill profit calculation)

total_prof = sum(b.get("totalProfit", 0) for b in bills)
# totalProfit already = (base_price - cost) for each bill

total_expenses = 0  # Sum of OPERATIONAL expenses only

net_profit = total_prof - total_expenses  # CORRECT - no double counting
```

Added comprehensive comments explaining:
- Why inventory costs are excluded
- How auto-generated expenses work
- Why net profit = gross profit - operating_only_expenses
